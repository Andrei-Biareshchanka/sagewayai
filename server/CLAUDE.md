# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with the **server** (backend) of SagewayAI.

> Coding style, naming, and TypeScript conventions: see **[CONVENTIONS.md](../CONVENTIONS.md)**.

## Stack

- Node.js + Express 5 + TypeScript 6
- Prisma 7 ORM + PostgreSQL
- Zod 4 for request validation
- bcryptjs + jsonwebtoken for auth (JWT access + refresh tokens)
- Resend for transactional email
- Vitest for tests

## Commands

```bash
npm run dev          # ts-node-dev hot reload on :3001
npm run build        # tsc → dist/
npm start            # node dist/index.js
npm test             # vitest run (single pass)
npm run test:watch   # vitest watch mode

# Single test file
npx vitest run src/path/to/test.ts

# Database
npx prisma migrate dev --name <migration-name>
npx prisma studio        # visual DB browser at :5555
npx prisma db seed       # seed topics + sample stories
npx tsc --noEmit         # type-check without building
```

## Project structure

```
src/
├── index.ts          # Express app entry point
├── routes/           # Route handlers grouped by resource
├── middleware/        # Auth, error handling, validation
├── services/         # Business logic (no Express types here)
└── lib/              # Prisma client, email, jwt helpers
```

## Prisma notes

Using **Prisma 7** — includes a `prisma.config.ts` file at the server root (generated by `prisma init`). Schema is in `prisma/schema.prisma`.

Always import the shared Prisma client instance — never instantiate `PrismaClient` inline:

```ts
// src/lib/prisma.ts
import { PrismaClient } from '@prisma/client';
export const prisma = new PrismaClient();
```

After any schema change: `npx prisma migrate dev --name <name>` then `npx prisma generate`.

## Request validation pattern

Use Zod to validate request bodies at the route level before passing to services:

```ts
const createStorySchema = z.object({
  title: z.string().min(1),
  content: z.string().min(1),
  type: z.enum(['PARABLE', 'SUCCESS_STORY']),
});

router.post('/stories', authenticate, async (req, res) => {
  const body = createStorySchema.parse(req.body);
  // ...
});
```

## Auth pattern

- Access token: short-lived JWT (15min), sent in `Authorization: Bearer` header
- Refresh token: long-lived JWT (7d), sent in httpOnly cookie
- `authenticate` middleware decodes access token and attaches `req.user`
- Role check middleware: `requireRole('ADMIN')`, `requireRole('PRO')`

## Error handling

All async route handlers must use try/catch. Use a central error middleware in `src/middleware/errorHandler.ts` — throw errors with a status code and message rather than calling `res.status().json()` inline everywhere.

## Data models (Prisma schema)

| Model | Key fields |
|---|---|
| `User` | `id`, `email`, `name`, `passwordHash`, `role` (USER/PRO/ADMIN), `emailSubscribed` |
| `Story` | `id`, `title`, `content`, `type` (PARABLE/SUCCESS_STORY), `status` (PUBLISHED/PENDING/REJECTED), `readTime`, `authorId` |
| `Tag` | `id`, `name`, `slug`, `storiesCount` |
| `Collection` | `userId` + `storyId` composite unique — join table for saved stories |

## Environment variables

Defined in `.env` (see `.env` file for values):
- `DATABASE_URL` — PostgreSQL connection string
- `JWT_SECRET` / `JWT_REFRESH_SECRET`
- `PORT` (default 3001)
- `CLIENT_URL` — CORS origin
- `RESEND_API_KEY` — for email sending
